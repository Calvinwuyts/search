ECFiddle Instructions

A simple interface for playing with Entity Collection retrieval

Overview

The purpose of the Entity Collection Fiddle tool (or 'ECFiddle' for short) is to allow easy exploration of the effect of different query strategies when querying our document collection with entities from the Entity Collection.

The interface is intended to be self-explanatory: the user enters text into the search box and picks an entity from the resulting autocomplete list. They then use the subsequent form controls to generate search clauses, and results are displayed in the right hand column.

Clauses and how they work

While the form controls themselves are simple, the way they interact to generate clauses may appear complex, at least at first glance.

Users can enter a maximum of four clauses, each of which may in turn be composed of up to five clauses itself.

Although these clauses are labelled 'subclauses' in the user interface, this name is perhaps a little misleading. The real point is that all the subclauses work together to form a single bracketed expression - thus allowing the articulation of complex queries using Boolean operators.

For example, supposing that you wanted to create a query that retrieved all records with 'Claude Debussy' listed as either a creator or a contributor - but only if this record had successfully been enriched with Debussy's Europeana EC identifier. This requires, in ECFiddle terms, two clauses to be expressed, one of which contains two subclauses:

(CREATOR:"http://data.europeana.eu/agent/base/146790" OR CONTRIBUTOR:("Claude Debussy" OR "Debussy, Claude")) AND (text:"http://data.europeana.eu/agent/base/146790")

Building and understanding these sorts of queries takes a certain amount of practice. In order to help the user understand the application logic, the generated query string is displayed above the list of results each time a query is submitted.

Search Controls

The controls for each clause are as follows:

  1. Radio buttons allowing the user to choose a clause operator: either 'AND' or 'OR'. The only exception to this rule is the first clause, which does not require an operator.
  2. A field name selector, allowing the user to pick which field the search will be applied upon.
  3. Radio buttons for 'mode': that is to say, whether the search will use the entity's Europeana ID (a URI) or a namestring.
  4. A textbox, indicating the value of the URI or namestring which will be used for the search.

When the user clicks 'Go', a search will be executed using the search string generated from these clauses, and the results displayed in the right-hand column. Alternatively, the user can clear the search using the 'Clear' button.
